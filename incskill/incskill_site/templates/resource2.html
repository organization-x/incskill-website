Variables and Constants

Variables are basically placeholders that can remember certain values and can be used later in the program. To understand this, consider an example. I want to solve a problem that has two steps.

Step 1. Compute 4+4

Step2. Output from step 1×2

There are two ways to do this problem. One way is that I intelligently transform this problem into (4+4)×2. Here, as our math skills tell us, first 4+4 will be computed, and then multiplied by 2. But, let's look at the second way of doing the exact same problem. Here we will take a rather non-intelligent way of doing it. We want to code one step at a time, and then we somehow want the program to reference the result from step1 in step2. Here is where variables come in. Read the code below carefully.

a = 4+4

a*2

If you try running this, you notice that it's exactly the same result. What is happening here is that the first statement consists of a variable "a". The "=" sign is an assignment operator which assigns a value to this variable and this value is mentioned to the right side of this "=" operator. This value is then remembered by the program and you can reference this later as happens in the second line of the code below.

Consider another example.

a = 2

b = 5

c = a + b

print(c*5)

Here, the print statement which prints a value to the console (remember the last week's resource?) gets executed and it needs to print c×5. So the program looks at what is c assigned to. c is assigned to a + b. But now what is a assigned to? 2. What about b? 5. And so, the value of c is 2+5 or 7. So, it prints 7 × 5 = 35.

One thing to remember with variables is that these values can be reset, which is why they are known as variables. Consider the updated example.

a = 2

b = 5

c = a + b

print(c*5)

c = 3

print(c*5)

While the first print will give you the result as we discussed, what do you think the second one will give? 15. This is because by assigning c to another value, we effectively reset the old value.

Constants different from variables exactly in this aspect. Constants are meant to be constants; they shouldn't be modified later in the scope of the program or program block. Other than that, there is no difference between variables and constants.

Conditionals

Let's now discuss conditional statements. One of the overarching ideas of programming is that we want computers to do something based on specific logic. A crucial component of logic is the logical flow "if A happens, do X. If B happens, do Y. Else, do Z". Somehow this needs to be translated into computer code. This is where conditionals come in the picture.

The way we work with conditionals in Python is primarily through the "if"...."else" construct. But before we discuss that, it's important to understand logical blocks. Logical blocks are essentially lines of code that go together. for instance, in this case, we may want five lines of code to be executed if A is true, otherwise five other lines of code. Now, these five lines of code that need to be executed constitute one block, while the other five lines constitute another block. The way these blocks are represented in Python is essentially through indentation. Now, this can be one of the most tricky aspects of programming for someone using a screen reader, since screen readers aren't necessarily going to give you indentation information out of the box. And so, this is one of those aspects that may frustrate you a little bit in the beginning, but trust me, with practice, this becomes pretty easy.

Here is an example. Don't worry about the exact code, just understand how blocks are structured through indentation.

if a == 5: # No spaces

print("Hello") # This line is offset by four spaces which represents that this is the next level, i.e. the level within the "if" statement and hence the lines that should be executed if the if condition above is true

print("Hello2") # This line is also offset by four spaces, i.e. at the same level as the previous line which means that this is a part of the same logical group and should also be executed if the condition above is true

else: # this line has no proceeding spaces and hence is at the root level, i.e. the same level as the "if" condition

print("Hello") # This line is offset by four spaces which represents that this is the next level, i.e. the level within the "else" statement and hence the lines that should be executed if the if condition above is not true

print("Hello2") # This line is also offset by four spaces, i.e. at the same level as the previous line which means that this is a part of the same logical group and should also be executed if the condition above is false

You can also use tabs for indentation since that can be easier to work with, although eventually, industry prefers spaces with four spaces representing each level of the block.

Before we discuss the "if" conditions specifically, we need to introduce some logical and conditional operators. These operators help us establish conditions and logic.

Conditional operators: < (checks if the left side is less than the right side), > (checks if the left side is greater than the right side), == (checks if the left and right are the same), <= (LHS less than or equal to RHS), >= (LHS is greater than or equal to RHS), <> (LHS not equal to RHS)

For practice, try out a few of the following expressions in the interactive mode.

4 == 4

4 < 5

5 > 1

5 > 6

For each of them, you get a "True" or "False" output representing if the condition you specified is true or false. "True" and "False" are keywords in Python that represents the truth of a statement.

Logical operators help combine evaluation of multiple expressions, thereby helping us to evaluate more complex expressions. For instance, what if we want to execute something if a is less than 5 and b is greater than 4. This is where logical operators come in.

There are three primary logical operators.

"And" operator: evaluates to true only when LHS and RHS both are true

"Or" operator: evaluates to true when either LHS or RHS is true

"Not" operator: Evaluates to true when the expression under evaluation is not true, i.e. it reverses the truth of the statement

Here are a few examples.

4==4 and 5==5 #True

4<5 and 5<6 #True

4==4 and 5==8 #False

4==4 or 5==8 #True

not(4 == 4) #False

Given this knowledge, let's now understand the "if...else" statement. The "if" statement helps us specify conditional flows (i.e. do A if X is true, B if Y is true or Z otherwise). Here is how it works in code:

if condition1:

Do this # Notice the four spaces representing an indentation level; this is very important

elif condition2:

Do this instead

elif condition3:

Do this

else:

Do this

Here is an example.

if a == 5:

print("hello world")

elif a == 7:

print("hello world 7")

elif a == 10:

print("hello world 10")

else:

print("Hello world 14")

This program will print "hello world" if the value of a is 5, "hello world7" if the value of a is 7, "hello world10" if the value of a is 10 or "hello world14" otherwise. There can be as many conditions as we want as part of this construct, and the conditions or the executing statements can be as complex as we want.

This brings us to the end of this tutorial. But, before we finish, one quick note about comments. You might have noticed that "#" has been used at various instances throughout this resource in the code. "#" is used to denote an inline comment, i.e. anything after the "#" sign is meant to be a comment for the reader and is ignored by the interpreter when executing the program. Similarly, if we have long comments spanning multiple lines, then we use """. Here is an example.

"""

This is an example of a multiline comment. This comment

is very large and will not be executed and will be completely ignored. This is

only for the benefit of the reader to help them understand what the code below this comment does.

"""
