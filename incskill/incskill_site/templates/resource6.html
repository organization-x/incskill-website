Week 6: Functions


A function is a logical group of code that is meant to accomplish a specific task. This is important because as you write more code, your code can be all over the place and it may become difficult to understand and debug (i.e. identify errors and fix them). Functions help you organize your code better by grouping your code into chunks that belong together and do a specific task. For instance, there can be a function to reverse a string, there can be yet another to calculate the No. of letters in a string, yet another to calculate the marks of a student based on a specific formula etc. As you write more code and get experienced, you determine what should constitute a function. A function has a name and zero or more arguments (also known as parameters) that it takes in as an input to perform the task. The function may return a value after it has performed its logic, or it may not return anything at all.

Here is how you define a function.

def <functionName>(<comma-separated list of arguments>):

<Function logic>

The logic of the function is indented just as how the logic within a loop or a conditional is indented. We have already seen many functions, except that we never discussed them as functions. For instance, print() is a function which prints to the console with the first argument being the content that should be printed. len() is a function that takes in an argument (such as a list) and returns the length of it.

Once a function has been defined and implemented, it can be called anywhere in the program. The following is an example of a function with and without parameters and how it is later called.

"""

addFive is a function which takes in a list and adds 5 to all of the elements and then returns the list.

"""

def addFive(l):

for i in range(len(l)):

l[i] = l[i] + 5

return l

# This function can then be called later in the program as follows.

l = [1, 5, 5, 8, 10]

l = addFive(l) #All integers will be increased by 5

Now consider a function without any parameter.

def logWarning():

print("The program has hit a warning")

# Let's assume that we want to print a warning message when a variable called a is 0

if a == 0:

logWarning()

Some functions (such as print) are available to us by default. But, there are several functions that are available to us but require certain libraries to be imported in the beginning of the program. For instance, if you remember the math operations in the first resource (including such functions as math.log), they required the math library to be imported as follows.

import math

#Later in the program

math.log(value for which log is required)

Thus, as you develop your skills, you will come across several functions, and you will need to know what libraries you may need to import to use them. Google will be your best friend when doing so.

File Handling

Now that you understand functions, let's move on to file handling, i.e. reading from files, writing to files etc. The open() function is a function that is available to you by default that helps you work with files. It's syntax is as follows.

f = open(<nameOfFile>, <mode>)

In other words, it takes two arguments-the name of the file and the mode. The mode argument can take two values "r" and "w" corresponding to read and write with the default value "r". Here is an example of this function in use.

f = open("listOfStudents.txt")

In this case, it is going to read lines from listOfStudents.txt file on your computer, and store the contents in variable f. You can then iterate as follows.

for line in f: #This is going to read line by line

<Do what you want to do with each line. Note this logic is indented because you are in a for loop.>

Note here that the file name specified as the first argument needs to be in the same directory as the code, or you need to provide an absolute path to the file. e.g. Let's say your code lives in the C:\documents directory. Then, the file that you want to read should be in the same directory, or should be referred to with the complete path (e.g. C:\<some other folder>\<some other subfolder>\<fileName>). Also, note that "\" is a special character use for escaping in Python, hence you need to replace "\" with "\\" in your file path for it to work.

Further, you can write to a file as follows.

f = open("fileToBeWritten.txt", "w") #Note the second argument here. This suggests mode writing.

f.write("Hello") #This writes Hello to the file

f.write("Hello2") #This writes Hello2 to the file right after Hello

The output of this file will be:

HelloHello2

Notice that these are merged. This is a good time to introduce newline character or "\n". This helps introduce a new line. So, you can write the above code as follows.

f = open("fileToBeWritten.txt", "w")

f.write("Hello\n") #This writes Hello with a new line after it to the file

f.write("Hello2") #This writes Hello2 to the file

Output:

Hello

Hello2

Project

Given all of this knowledge, you are now equipped to work on your first project. As part of this project, you will develop a two-player word game. The game uses a large list of English words, jumbles these words, and asks the user to reorder the characters to identify the correct word. Each user gets two attempts and scores points in proportion with the length of the word. Each game lasts five words for both the users at the end of which the winner is displayed. Here are some instructions to get you started.

  • You can start off with a list of the 3,000 most common words in English available at https://www.ef.edu/english-resources/english-vocabulary/top-3000-words/. Copy over these words to a text file with each word on a line.
  • Using file handling and list operations, read these words into a list.
  • Google the random function in Python (I'm deliberately not providing details to get you into the habit of finding, reading and understanding function documentation online) to randomly pick numbers from this list and jumble it. When you are jumbling the word, remember that a string is essentially a list.
  • Put it all together with your understanding of conditionals and loops to design the overall flow.
  • Use a dictionary to track the score of both the players and ultimately print the winner.
